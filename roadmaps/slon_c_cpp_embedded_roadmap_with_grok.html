Слон №1 — C/C++ + Embedded

Дорожная карта на 6 месяцев (теория + практика) с учётом книги «Грокаем алгоритмы».

Общая стратегия

  • Чередовать теорию и практику каждую неделю.
  • Мини-проекты на 50–150 строк каждую неделю.
  • Постепенное наращивание сложности: C → C++ → Linux → Embedded.
  • Добавлен алгоритмический блок с «Грокаем алгоритмы» (Адитья Бхаргава) как разогрев перед Седжвиком.

План по месяцам

НеделяТеорияПрактикаРезультат
1–2Шилдт по C: типы данных, условия, циклы, функции. Консольные программы: калькулятор, массивы, ввод-вывод. Базовый синтаксис, умение писать маленькие программы на C.
3–4Указатели, массивы, строки, структуры. Побитовые операции. Парсер строки в структуру, работа с файлами. Понимание работы с памятью и структурированных данных.
5–6Страуструп: классы, ООП, STL. C++20: auto, lambda, smart pointers. Мини-проекты с классами: банковский счёт, склад товаров. Переход от C к современному C++, владение базовыми STL-контейнерами.
7Грокаем алгоритмы: Big-O, бинарный поиск, рекурсия, быстрая сортировка, хеш-таблицы. Реализация: binary_search, quick_sort, простая hash table. Сравнение с std:: версией. Понимание базовых алгоритмов и сложности, навыки бенчмаркинга.
8–9Седжвик: сортировки, поиск. Структуры данных: стек, очередь, список. Собственные реализации + тесты и бенчи. Библиотека алгоритмов/структур с тестами.
10Грокаем алгоритмы (микро-спринт): BFS, Дейкстра. Реализация BFS (лабиринт), Dijkstra (min-heap). Понимание графов и поиска путей.
11–12Linux-программирование (Секунов): файлы, процессы, сокеты. Утилита фильтрации логов, чат по сокетам, make/cmake. Базовые навыки системного программирования.
13–14Arduino (Блум/Иго/Бокселл) — 2–3 проекта. Мигание + кнопка, датчик на OLED, UART/Bluetooth. Быстрый старт в embedded, понимание high-level API.
15–16«Голое железо»: STM32 или ESP32. GPIO, прерывания, UART, I2C, SPI. Регистровый доступ, LL/HAL, минимальные драйверы. Навык низкоуровневой работы с MCU.
17–18FreeRTOS: задачи, очереди, таймеры. Приложение: датчик → UART/Wi-Fi → SD-карта. Полноценный embedded-проект с RTOS.

Практические советы

  • Вести все проекты в одном репозитории с CI (сборка, тесты, бенчи).
  • Для алгоритмов — сравнивать свои реализации со стандартными библиотеками.
  • Для Embedded — документировать схему подключения, тайминги, энергопотребление.
